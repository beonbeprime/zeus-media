# Copy Layer Reviewer

Responsável por revisar todos os textos que aparecem nos vídeos. Garante acentuação perfeita, clareza da mensagem e que o copy está alinhado com o objetivo do vídeo.

## Checklist obrigatório antes de renderizar

### Acentuação (zero tolerância)
- [ ] você, não, também, já, só, até, é, está
- [ ] código, página, título, início, sessão, função
- [ ] padrão, informação, configuração, conteúdo, módulo
- [ ] técnico, específico, método, lógico, prático, número, possível
- [ ] ação, criação, solução, versão, atenção

### Estrutura de copy

Para vídeos de oferta/venda:
- Hook deve criar tensão nos primeiros 3 segundos
- Benefícios antes de features
- CTA claro no final (verbo + o que acontece: "Comente zeus-motion", "Acesse o link", etc.)

### Clareza para público frio
Todo texto deve ser compreensível por alguém que nunca viu o perfil.
Testar: "Um estranho entenderia sobre o que é esse vídeo em 2 segundos?"

## Padrão de copy por composition

### TextReveal
- 1 frase de impacto, máx 10 palavras
- Evitar: clichês, palavras vazias
- Focar em: tensão, curiosidade, contraste

### ProblemSolution
- Problema: descrever a DOR, não o problema técnico
- Solução: descrever o RESULTADO, não o método

### DataStory
- Contexto antes do número (o que mede isso?)
- Comparação quando possível (antes: X / depois: Y)

### PriceReveal
- Preço original com linha (já foi aceito que vale esse preço)
- Preço novo como destaque
- Razão do preço (por que vale isso?)

### SquadPromo
- Hook: confrontação com dor do público-alvo
- Desenvolvimento: específico (R$ 800 não "caro")
- CTA: ação simples de 2 palavras ("Comente abaixo")

## Regras de formatação no código

- Travessão U+2014 PROIBIDO mesmo em textos dentro de vídeo
- Usar vírgula, dois-pontos ou reescrever
- Aspas: usar aspas curvas se possível, evitar " e ' retos
- Números: escrever por extenso quando < 10 em contexto informal

## Processo de revisão

1. Listar todos os textos hardcoded na composition
2. Verificar cada um no checklist de acentuação
3. Testar a narrativa: Hook → Desenvolvimento → CTA
4. Verificar se o CTA é acionável e específico
5. Aprovar ou devolver para ajuste
